## WS-1 Status Report

### Tasks completed

- WS-1.1 — Added header-extraction middleware (`packages/omnimind-api/src/middleware/agent-context.ts`). Reads `x-agent-id`, `x-tenant-id`, `x-source-weight`; falls back to `Agent` lookup by `apiKeyHash(x-api-key)` when headers absent. Attaches to `req.agentContext`. Wired into `index.ts` immediately after `apiKeyAuth`.
- WS-1.2 — Extended `memory.service.ts` signatures: `createMemory` / `updateMemory` / `getMemory` / `searchMemories` / `archiveMemory` accept optional `agentContext` and write `agentId`/`tenantId`/`sourceWeight` from it. The dedup branch in `createMemory` now passes context through to the supersede `updateMemory` call (Bug #3 fix). Backward-compat shim (`resolveContextAndPrisma`) lets legacy `(userId, input, prisma)` callers keep working unchanged — verified against existing unit tests in `packages/omnimind-api/tests/unit/services/memory.service.test.ts`.
- WS-1.3 — Wired routes:
  - `memories.routes.ts` — every handler passes `req.agentContext` to the service.
  - `memories.routes.ts` `POST /search-similar` — previously dropped `tenantId` entirely; now scopes the raw cosine query by `req.agentContext.tenantId`. Bug #2 fix.
  - `admin.routes.ts` — `/stats`, `/agents`, `/audit`, `/memories` default to tenant-scoped via a new `resolveTenantFilter(req)` helper. `?includeAllTenants=true` opts into cross-tenant view.
  - `context.routes.ts` — forwards `req.agentContext.tenantId` into the retrieval scope.
- WS-1.4 — MCP client (`packages/omnimind-mcp/src/lib/client.ts`): the original code-review claim was wrong — headers were NOT being sent. Added `setAgentHeaders` plus `AgentHeaders` interface; every outbound `request()` now carries `x-agent-id`, `x-tenant-id`, `x-source-weight` when set. `server.ts` binds the headers from the resolved AgentContext at startup.
- WS-1.5 — Retrieval-layer tenant filter applied to `semantic-search.ts`, `fulltext-search.ts`, `trigram-search.ts`, `structured-filter.ts`. Each now accepts `tenantId` and `includeAllTenants` in its options. Default when neither is supplied: return `[]` (safer than silent cross-tenant leakage). Updated `context-assembler.service.ts` to forward the scope.
